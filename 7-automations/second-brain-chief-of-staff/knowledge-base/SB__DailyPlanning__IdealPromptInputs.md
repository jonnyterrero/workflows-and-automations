## Context
- Purpose: standardize what information is provided each day for reliable planning output.
- Use this as the morning checklist before asking for a daily plan.

## Key Ideas
- Required input block:
  - Date and day type (class day, lab day, weekend).
  - Available work windows with start/end times.
  - Top deadlines in next 14 days.
  - Current energy level (1-5) and sleep quality.
  - Active blockers.
  - Non-negotiables (training, liturgy, meetings).
- Optional but high-value:
  - Current stressors/distractions.
  - Course/project confidence ratings.
  - Yesterday’s carry-over tasks with reason unfinished.

## Decisions
- Ask for outputs in strict format: Situation, What matters most, Sequence, Concrete actions, Risks.
- Require explicit deferrals: what will not be worked on today.
- End by identifying the first 30-minute action that starts momentum.

## Constraints
- Bad input quality leads to generic planning.
- Overestimating available time causes cascading misses.
- Ambiguous tasks increase procrastination probability.
