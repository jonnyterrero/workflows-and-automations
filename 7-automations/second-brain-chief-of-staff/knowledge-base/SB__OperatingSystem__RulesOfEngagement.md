## Context
- Problem: Too many parallel tracks create context switching and shallow progress.
- Need: A strict rule set for how to accept, schedule, and finish work.

## Key Ideas
- Rule 1: No plan is valid without constraints (time, energy, dependencies).
- Rule 2: Every week has one defining academic outcome and one defining project outcome.
- Rule 3: Work in sequence, not mood. Execute highest leverage block first.
- Rule 4: Tasks must be verb-first and completion-verifiable.
- Rule 5: If workload exceeds capacity, cut scope explicitly instead of silently slipping.
- Rule 6: No new commitments without identifying trade-offs.
- Rule 7: End each day with a written reset for tomorrow’s first block.

## Decisions
- Use 90-minute deep-work blocks for cognitively heavy tasks.
- Cap active projects to 3 (excluding coursework).
- Maintain a rolling blocker list and clear one blocker daily.
- Escalate unresolved blockers >72 hours to advisor/teammate/professor.

## Constraints
- Class timetable and lab access windows are fixed.
- BJJ and lifting sessions consume physical/mental recovery bandwidth.
- Communication overhead (email/Slack) can consume mornings if unbounded.
